Method detail
onClose()method
public function onClose(connection:Connection):void

onStop is called when all consumers have finished processing, sessions have closed, and the WebSocket connection is shutdown. No further messages can be sent or received. This listener is fired in response to calling connection.close().

Parameters
connection:Connection
onConnect()method 
public function onConnect(connection:Connection):void

onConnect is called when the underlying WebSocket connection has been established, and the client has authenticated with the Kaazing Gateway. This listener is fired in response to a call to JMSConnectionFactory.createConnection().

Parameters
connection:Connection
onStart()method 
public function onStart(connection:Connection):void

onStart is called when the connection has completed initialization tasks required to subscribe and consume messages, in response to calling connection.start(). An exception is thrown if stop() is called while starting before this listener is fired.

Parameters
connection:Connection
onStop()method 
public function onStop(connection:Connection):void

onStop is called when the message flow from the provider has stopped, and messages will no longer be delivered to consumers, in response to calling connection.stop(). To start the flow of messages, call connection.start() after this listener has fired. An exception is thrown if start() is called while stopping before this listener is fired.

Parameters
connection:Connection
