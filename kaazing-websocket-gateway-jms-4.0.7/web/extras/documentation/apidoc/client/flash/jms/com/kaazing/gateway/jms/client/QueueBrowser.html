Interfacepublic interface QueueBrowser

A client uses a QueueBrowser object to look at messages on a queue without removing them. Not implemented.

The getEnumeration method returns a MethodEnumeration that is used to scan the queue's messages. It may be an enumeration of the entire content of a queue, or it may contain only the messages matching a message selector.

Messages may be arriving and expiring while the scan is done. The JMS API does not require the content of an enumeration to be a static snapshot of queue content. Whether these changes are visible or not depends on the JMS provider.

A QueueBrowser can be created from either a Session or a QueueSession.

close()method
public function close():void

Closes the QueueBrowser.

Since a provider may allocate some resources on behalf of a QueueBrowser outside the immediate environment, clients should close them when they are not needed. Relying on garbage collection to eventually reclaim these resources may not be timely enough.
