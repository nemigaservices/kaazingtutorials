Class JmsConnectionFactory

JmsConnectionFactory provides an interface for creating new JMS connections via a WebSocket connection to a JMS provider.

Creates a new JmsConnectionFactory to create connections to the target url specified. A WebSocket is used to establish the connection. The url should specify a protocol such as ws: or wss:. For example: ws://localhost:8000/jms

createConnection

ConnectionFuture createConnection(username, password, clientID, callback)
    Creates the actual JMS Connection via WebSocket, using the username and password specified as credentials to authenticate with the Gateway, and using the given clientID to distinguish the durable subscribers for this connection from those of other users.

    Once the callback is invoked, client applications should use ConnectionFuture.getValue() to fetch the actual connection.

    Parameters:
      String username - The username for authentication or null. (Optional)
      String password - The password for authentication or null. (Optional)
      String clientID - Client identifier. (Optional)
      This is combined with durable subscriber names, using the format specified for "jms" service parameter "client.durable.name.format", to make them unique for the current user.
      Function callback - The callback to invoke when the connection has been created or an exception is thrown.
    Returns:
      ConnectionFuture
